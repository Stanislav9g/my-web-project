🚀 Актуальность темы

Веб-технологии — основа современного мира. Понимание того, как устроены сайты, необходимо каждому пользователю. Мой проект делает процесс обучения наглядным и простым через систему аналогий.

🎯 Цель и задачи

Цель: исследовать эволюцию технологий и создать обучающий сайт.

  • Изучить теорию Frontend и Backend;
  • Раскрыть суть API через аналогии;
  • Исследовать причины отказа от Adobe Flash;
  • Разработать авторский веб-ресурс.

🛠 Методы работы

  • Теоретический: анализ документации;
  • Сравнительный: Flash против HTML5;
  • Практический: написание кода и верстка;
  • Статистический: диагностика знаний.

🖼 Frontend — зал ресторана

Это визуальная часть сайта. В работе я использовал HTML5 для структуры и CSS3 для стилей. Это всё, на что нажимает и на что смотрит пользователь.

🧑‍🍳 Backend — кухня

Скрытая логика сайта. Здесь обрабатываются данные, хранятся пароли и происходит работа с базами данных на сервере.

🤵 API — официант

Связующее звено. API передает запросы от пользователя на кухню (сервер) и приносит ответ обратно в зал (браузер).

⚠️ Полный отказ от Adobe Flash

Технология стала «тупиковой» из-за уязвимостей в безопасности, медленной работы и высокого потребления энергии. Её заменил открытый стандарт HTML5.

💻 Этапы разработки

Код писался в VS Code. Проект проходил стадии проектирования интерфейса, написания скриптов на JS и тестирования на разных устройствах.

📦 Стек и деплой

Для контроля версий использовался GitHub. Сайт опубликован через сервис GitHub Pages, что делает его доступным по ссылке из любой точки мира.

О проекте

Этот ресурс создан в рамках индивидуального проекта для наглядного объяснения основ веб-технологий. Здесь теория встречается с практикой.

Понятные аналогии

Frontend

Визуальная часть. Всё, что вы видите сейчас на экране — это фронтенд.

Backend

Серверная логика. Скрытые вычисления и базы данных.

API

Курьер или официант. Передает данные между частями сайта.
